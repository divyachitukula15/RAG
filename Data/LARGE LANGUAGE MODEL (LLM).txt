LARGE LANGUAGE MODEL (LLM)
A Large Language Model is a neural network trained on vast amounts of text data to understand and generate human-like text. It works by predicting the most likely next word in a sequence based on patterns learned during training. LLMs process text as tokens and use transformer architecture to understand context and relationships between words.
How It Works
LLMs process text through transformer architecture, converting words to tokens, applying attention mechanisms across multiple layers, and predicting the next most probable token based on learned patterns.
 
GENERATIVE AI (GENAI)

Generative AI encompasses various AI systems that can create new content across multiple modalities - text, images, audio, video, and code. It uses different model types like LLMs for text, diffusion models for images, and GANs for various creative tasks. GenAI is the broader category that includes LLMs as one of its components.
How It Works
GenAI uses specialized pipelines for different content types, each employing appropriate neural architectures (LLMs for text, diffusion models for images, etc.)  to transform prompts into creative outputs.

AI AGENT
An AI Agent is an autonomous system that combines LLMs with external tools and APIs to perform specific tasks. Unlike simple chatbots, agents can take actions in the real world - they can send emails, fetch data, fill forms, and interact with various services. They react to environmental changes and can complete multi-step processes independently.
How It Works
AI Agents break down tasks into actionable steps, select appropriate tools, execute actions in the environment, monitor results, and adapt their approach until the task is completed.

AGENTIC AI
Agentic AI represents the most advanced form of AI agents with sophisticated capabilities like long-term memory, goal persistence, multi-step reasoning, and self-reflection. These systems can maintain context across extended interactions, learn from experiences, adapt their strategies, and work toward complex goals over time. They're designed for tasks requiring sustained attention and planning.

How It Works
Agentic AI combines long-term memory, multi-step reasoning, and self-reflection to pursue complex goals autonomously. It continuously learns, adapts strategies, and maintains persistence across extended timeframes.

PFAS FRAMEWORK
The PFAS framework is a simple way to do AI-powered research in a structured and smart way.
It helps you avoid confusion and guides you step-by-step so AI gives better, accurate, and useful answers.
PFAS stands for:
P → Problem
	Define exactly what you want to know. This is where you ask a clear and specific question.
	If this step is weak, everything else becomes weak.
•	clear
•	specific
•	focused on a goal

F → Find
	Find the right information or sources.
•	it searches multiple places
•	finds relevant data
•	filters out useless content

A → Analyse
	Understand the information.
•	reads multiple sources
•	compares them
•	finds key insights, patterns, and differences
•	This is where real intelligence happens.

S → Scrape
	Collect and organize the data.
•	extracts useful data
•	formats it properly
•	presents it clearly (tables, summaries, etc.)

Easy analogy:
 Problem → What do you want?
 Find → Where to get it?
 Analyse → What does it mean?
 Scrape → Organize it neatly

LLM
	A Large Language Model is a neural network trained on vast amounts of text data to understand and generate human-like text. 
	It works by predicting the most likely next word in a sequence based on patterns learned during training. 
	LLMs process text as tokens and use transformer architecture to understand context and relationships between words.
 
Key Concepts Explained Simply
Tokens: These are the atomic units of language for an LLM. Instead of seeing whole words, the model breaks text into smaller chunks called tokens—which can be words, parts of words, or even single characters. In English, 100 tokens are roughly 75 words, i.e 1 token=3/4th word
Parameters: They are internal variables (like weights) that the model adjusts during training to learn the rules of language and logic. 
parameters
Context Window: This is the model's working memory. It is the maximum number of tokens the model can see and consider at one time when generating a response. If a conversation gets too long, it might forget the beginning because it has moved out of this window.
Training (Pre-training): The initial phase where the model reads vast amounts of data (like Wikipedia and the internet) to learn grammar, facts, and reasoning.
Fine-tuning: The process of taking a pre-trained model and giving it extra training on a smaller, specific dataset (like medical or legal texts) so it becomes an expert in that area.
Inference: This is the live stage when you actually use the model. It takes your input (prompt) and generates an output based on what it already knows.
Hallucination: When an LLM generates information that sounds very confident and plausible but is actually incorrect or completely made up.

LLM Architecture: The Transformer
Modern LLMs are built on an architecture called the Transformer, introduced in 2017. Its secret weapon is Self-Attention, which allows the model to process entire sentences at once and understand the relationships between words, even if they are far apart. 

Core Architecture Flow
Input Tokenization: Breaks your text into tiny chunks (tokens).
Embedding Layer: Converts those tokens into lists of numbers (vectors) that represent their meaning.
Positional Encoding: Adds tags to the numbers so the model knows the order of the words (e.g., the difference between dog bites man and man bites dog).
Transformer Blocks: The engine room where the model uses Self-Attention to weigh which words in a sentence are most important to the current context.
Output Layer: Converts the processed numbers back into probabilities to choose the best next token to show you.

 
RAG
	Retrieval-Augmented Generation (RAG) is a way to make AI answers more reliable by combining searching for relevant information and then generating a response. 
	Instead of guessing based only on old training data, it first finds useful data from external sources (like documents or databases) and then uses it to give a better answer.

•	Fetches up-to-date data and reduces incorrect or made-up answers
•	Works well with specialized data like medical or legal content
•	No need to retrain the model every time new data comes in
•	Can use user-specific data to give more relevant responses

Example
For example, a platform like GeeksforGeeks has its own large collection of coding articles and tutorials. If a user asks a question, instead of giving a general answer, a RAG-based system can:
1.	Search relevant articles from their dataset
2.	Pick the most useful content
3.	Generate an answer based on that specific information
This way, the response is more accurate, aligned with the platform’s content and actually helpful for the user.
 
Components of RAG:
1.	External Knowledge Source: Stores domain specific or general information like documents, APIs or databases.
2.	Text Chunking and Preprocessing: Breaks large text into smaller, manageable chunks and cleans it for consistency.
3.	Embedding Model: Converts text into numerical vectors that capture semantic meaning.
4.	Vector Database: Stores embeddings and enables similarity search for fast information retrieval.
5.	Query Encoder: Transforms the user’s query into a vector for comparison with stored embeddings.
6.	Retriever: Finds and returns the most relevant chunks from the database based on query similarity.
7.	Prompt Augmentation Layer: Combines retrieved chunks with the user’s query to provide context to the LLM.
8.	LLM (Generator): Generates a grounded response using both the query and retrieved knowledge.
9.	Updater (Optional): Regularly refreshes and re-embeds data to keep the knowledge base up to date.



MCP
	The Model Context Protocol (MCP) is an open-source standard that allows AI applications/ models to securely connect with external data sources and tools seamlessly. 
	Introduced by Anthropic on 25 November 2024. 
	It acts as a universal connector—often compared to a USB-C port for AI—enabling AI models to securely access files, databases, and APIs without custom integrations for every new tool.
	It access live information and perform actions beyond their static training data. 

Key Features (Primitives)
 
Servers share capabilities with clients through three primary primitives: 
1. Prompts (The Instruction Manuals)
These tell the AI how to act and what to say.
	LLM Guidance Templates: Pre-written instructions that give the AI a personality or a specific way to think (e.g., Act like a senior coder).
	Tool Usage Instructions: Clear steps that tell the AI exactly when and how to use a specific tool so it doesn't make mistakes.
	Resource Formatting: Instructions on how to organize the data it finds so it’s easy for you to read (like putting info into a table).
2. Resources (The Library of Data)
This is the knowledge the AI can look up. It allows the AI to see your private information safely.
	Documents & Files: The AI can read your PDFs, Word docs, or text files.
	Database: It can talk to professional data storage systems to find specific records.
	Private Knowledge: Any unique information that belongs to you or your company that wasn't part of the AI's original training.
3. Tools (The Action Centre)
This is where the AI does things instead of just talking.
	Function Execution: The AI runs small pieces of code to solve problems (like doing a complex math calculation).
	API Interactions: The AI talks to other apps (like sending a message on Slack or checking a weather website).
	Data Processing: The AI takes raw, messy information and cleans it up or analyses it.

Why It Matters
Scalability: It solves the  N*M integration problem. Instead of building custom code for every tool (N) and every AI model (M), developers only need to implement the Model Context Protocol once for each side.
Freshness: It provides on-demand access to real-time data, significantly reducing hallucinations caused by outdated training data.
Interoperability: Being an open standard, it allows different AI providers like Google Cloud and Cloudflare to use the same set of pre-built community servers found on GitHub. 

MCP Architecture
The Model Context Protocol (MCP) architecture is a three-part system designed to let AI models plug in to data and tools just like you plug a USB device into a computer. 
 
The 3 Core Components
The architecture is built on a Client-Server model with a Host overseeing everything: 
MCP Host (The Brain's Home)
	The actual application you use, like Claude Desktop, Cursor, or VS Code.
	It is the orchestrator. It holds the AI model (LLM) and decides when it needs to reach out to a tool. It manages security and asks for your permission before any action is taken.
MCP Client (The Translator)
	A small piece of software inside the Host application.
	It acts as the bridge. It maintains the connection to the server and translates the AI's natural language needs into structured code that the server can understand.
MCP Server (The Specialized Tool)
	A lightweight program that connects to a specific data source (like Google Drive, Slack, or a database).
	It exposes what it can do. It tells the client, I can read files, or I can search a database, and then performs those actions when asked. 
How Communication Works (The Wires)
	The protocol uses a Transport Layer to move messages back and forth. There are two main ways it does this: 
	STDIO (Standard Input/Output): Best for local tools running on your same computer. It’s fast and very secure because data doesn't leave your machine.
	SSE (Server-Sent Events): Used for remote tools in the cloud. It allows the server to push updates to the client in real-time. 


Example: Summarize Emails
Step 1: The Request (User Query)
You type Summarize my recent emails into an MCP Client Application (e.g., Claude Desktop). The application packages this text and sends it to the LLM.
Step 2: Intent & Tool Selection
The LLM analyses the request. It recognizes it cannot see your inbox, so it searches its list of connected capabilities. It selects the specific Tool (e.g., list_emails) provided by an email MCP server.
Step 3: The Client Call
The MCP Client takes that selection and sends a formal request over the Standardized Protocol to the MCP Server (the one specifically built to talk to your email provider).
Step 4: Server Execution (External API)
The MCP Server receives the command. It reaches out to the External API (like Gmail or Outlook), fetches the raw text of your recent emails, and brings that data back into the MCP environment.
Step 5: Context Injection
The MCP Client receives the raw email data from the server and feeds it back to the LLM. The LLM now has its original intelligence plus the specific context of your actual emails.
Step 6: Generation & Response
The LLM reads the emails, performs the summarization, and generates a natural language response. The MCP Client displays this final summary to you in the chat interface.




EVOLUTION
 
Phase 1: Basic LLMs (The Knowledge Only Phase)
Think of this like a smart person who is locked in a room with only their memories.
What it is: The AI uses only the information it learned during its training.
The Limit: It doesn't know what is happening in the world right now and can't use any apps or files. It can only talk about what it already knows.
Phase 2: Agentic Frameworks (The Custom Tools Phase)
Now, imagine that same person is allowed to use a few tools, but every tool has a different, complicated plug.
What it is: Developers started hand-wiring the AI to specific tools (like a search engine or a database).
Glue code is custom, manual programming written solely to connect and translate data between incompatible software components.
